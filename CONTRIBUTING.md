## 1. What is KlasseTavle?

An interactive fullscreen classroom board for Norwegian teachers (age 8–9 students). It is a **canvas** of draggable, resizable **widgets**. The teacher opens it on a projector. Students see the screen.

- All UI text is in **Norwegian**.
- Everything is client-side React. No server rendering.
- State lives in a Zustand store, persisted to localStorage, optionally synced to Supabase.

**Live URL:** https://klassetavle.no
**Repo:** https://github.com/BlastBlastBlast/klassetavle
**Tech stack:** React 19 · TypeScript 5.9 · Vite 8 · Tailwind CSS 4 · Zustand 5 · Supabase JS 2

## 4. State Management — boardStore.ts

> **This is the most important file.** Read it completely before adding any feature.
> Path: `src/store/boardStore.ts`

### Core types

```typescript
type WidgetType =
  | 'trafikklys' | 'tekst' | 'bilde' | 'klokke' | 'terning'
  | 'timer' | 'snurrehjul' | 'tegning' | 'bakgrunn' | 'gestu'
  // ADD YOUR NEW TYPE HERE — exact lowercase string, no spaces

interface Widget {
  id: string          // uuid — generated by crypto.randomUUID()
  type: WidgetType    // determines which component renders
  x: number           // position on canvas (px from left)
  y: number           // position on canvas (px from top)
  width: number       // current width in px
  height: number      // current height in px
  zIndex: number      // stacking order — managed by bringToFront()
  data: Record<string, unknown>  // widget-specific state — YOU define the shape
}

interface Canvas {
  id: string
  name: string
  widgets: Widget[]
  background: string  // CSS value: color, gradient, or url(...)
  maxZ: number        // highest zIndex in use on this canvas
}
```

### Actions you will use

| Action | Signature | When to use |
|--------|-----------|-------------|
| `addWidget` | `(type: WidgetType) => void` | Toolbar button click |
| `removeWidget` | `(id: string) => void` | Widget close button |
| `updateWidget` | `(id, Partial<Widget>) => void` | Move / resize (position/size only) |
| `updateWidgetData` | `(id, Partial<data>) => void` | **Widget internal state changes** |
| `bringToFront` | `(id: string) => void` | On widget click/focus |
| `setBackground` | `(bg: string) => void` | Bakgrunn widget only |
| `setGlobalGesture` | `(id: string \| null) => void` | Gestu widget only |

### DEFAULT_SIZES

Every WidgetType **must** have an entry in `DEFAULT_SIZES`:

```typescript
const DEFAULT_SIZES: Record<WidgetType, { width: number; height: number }> = {
  trafikklys:  { width: 160, height: 340 },
  tekst:       { width: 300, height: 200 },
  // ... add yours here
}
```

If your type is missing from DEFAULT_SIZES, `addWidget` will crash.

### How widget data (internal state) works

Your widget stores all its own state inside `widget.data`. Example:

```typescript
// Reading data inside your widget component:
const value = (widget.data.myField as string) ?? 'default'

// Writing data (use updateWidgetData — it shallow-merges):
const { updateWidgetData } = useBoardStore()
updateWidgetData(widget.id, { myField: newValue })
```

`updateWidgetData` does a **shallow merge** — you only pass the fields that changed.
Do **not** call `updateWidget` for internal state — that is for x/y/width/height/zIndex only.

### Persistence

The entire store is persisted to **localStorage** automatically on every change (key: `klassetavle-board`).
Supabase sync is triggered manually by the user via the AuthBar. You do not need to call sync from widgets.

## 5. WidgetWrapper — How Drag & Resize Work

> Path: `src/components/canvas/WidgetWrapper.tsx`

**Every widget is wrapped by WidgetWrapper.** You do not manage drag/resize yourself.

```tsx
// Pattern used in every widget:
return (
  <WidgetWrapper widget={widget} title="Min Widget" minWidth={200} minHeight={150}>
    {/* Your widget UI here */}
  </WidgetWrapper>
)
```

### Props

| Prop | Type | Default | Purpose |
|------|------|---------|---------|
| `widget` | `Widget` | required | The widget object from store |
| `title` | `string` | `''` | Shown in the drag handle header |
| `minWidth` | `number` | `120` | Minimum resize width |
| `minHeight` | `number` | `80` | Minimum resize height |

### How drag works (important for contributors)

Drag uses **pointer events** (not react-draggable — that was removed due to React 19 incompatibility). Position is stored in `widget.x` / `widget.y` in the store. During drag, the DOM is mutated directly via `style.left` / `style.top` for performance (no React re-renders per frame). On drag end, `updateWidget` commits to the store.

**Do not add `position: absolute` or `transform` to your widget's root element** — WidgetWrapper handles positioning.

### How resize works

`react-resizable` ResizableBox wraps the content. Resize handles are hidden by default, visible on widget hover (`.widget-container:hover .react-resizable-handle` in `index.css`). On resize stop, `updateWidget` commits new width/height to store.

## 6. Adding a New Widget — Step by Step

Follow these 5 steps **in order**. Do not skip any step.

### Step 1 — Add the type to boardStore.ts

File: `src/store/boardStore.ts`

```typescript
// Find this type union and add your type:
type WidgetType =
  | 'trafikklys' | 'tekst' | ... | 'gestu'
  | 'minnavn'   // ← add here, lowercase, no spaces
```

```typescript
// Find DEFAULT_SIZES and add an entry:
const DEFAULT_SIZES: Record<WidgetType, { width: number; height: number }> = {
  // ... existing entries ...
  minnavn: { width: 300, height: 250 },
}
```

### Step 2 — Create the widget component

File: `src/components/widgets/MinNavn.tsx`
Copy the template from `docs/widget-template.tsx` and fill it in.

Key rules:
- Component receives `{ widget: Widget }` as its only prop.
- Root element must be `<WidgetWrapper widget={widget} title="...">`.
- All UI text must be in **Norwegian**.
- All internal state lives in `widget.data` — read via cast, write via `updateWidgetData`.
- Do **not** use React `useState` for data that should persist — use `updateWidgetData` instead.
- `useState` is fine for purely visual/transient state (hover, animation phase, open/closed panel).

### Step 3 — Register in WidgetRenderer.tsx

File: `src/components/canvas/WidgetRenderer.tsx`

```typescript
import { MinNavn } from '../widgets/MinNavn'

// Add to the switch statement:
case 'minnavn': return <MinNavn widget={widget} />
```

### Step 4 — Add toolbar button in Toolbar.tsx

File: `src/components/canvas/Toolbar.tsx`

```typescript
// Find the tools array and add:
{ type: 'minnavn', emoji: '🎯', label: 'Min Navn' },
```

No other changes needed in Toolbar — the button click already calls `addWidget(type)`.

### Step 5 — Verify

```bash
npm run build   # must complete with 0 errors, 0 TypeScript errors
npm run dev     # open browser, click toolbar button, widget appears and is draggable
```

## 7. Widget Design Conventions

### Layout inside WidgetWrapper

```tsx
<WidgetWrapper widget={widget} title="Tittel" minWidth={200} minHeight={150}>
  <div className="flex flex-col h-full p-3 gap-3">
    {/* Controls / settings at top */}
    <div className="flex gap-2 flex-wrap">...</div>

    {/* Main content fills remaining space */}
    <div className="flex-1 flex items-center justify-center">...</div>
  </div>
</WidgetWrapper>
```

### Colors and styling

- **Background**: `bg-white/10` or `bg-black/20` — semi-transparent
- **Buttons**: `rounded-lg px-3 py-1.5 text-sm font-medium` + a bg color
- **Active state**: `ring-2 ring-white/60`
- **Text**: `text-white` (all widgets sit on dark/colorful backgrounds)
- **Icons**: use `lucide-react` — `import { IconName } from 'lucide-react'`
- **Animations**: defined in `src/index.css` — classes: `animate-float-in`, `animate-bounce-in`, `animate-spin-slow`

### Norwegian text examples

```
"Legg til"   → Add
"Fjern"      → Remove
"Start"      → Start
"Stopp"      → Stop
"Lagre"      → Save
"Tilbakestill" → Reset
"Velg"       → Select/Choose
```

### Adding new CSS animations

Add to `src/index.css`:

```css
@keyframes my-animation { from { ... } to { ... } }
.animate-my-animation { animation: my-animation 0.4s ease forwards; }
```

## 8. Supabase Integration

> Only add Supabase integration if the feature requires **server-side persistence** that must survive across devices/sessions. For single-session state, localStorage (the default) is sufficient.

### Pattern for a new Supabase table

1. Add SQL to `SUPABASE_INTEGRATIONS.md` (keep a record)
2. Run the SQL in Supabase → SQL Editor
3. Add service functions to `src/lib/boardService.ts` (follow existing CRUD patterns)
4. Call from the widget or store action — always check `if (!supabase) return null` first

### Supabase client

```typescript
import { supabase } from '../../lib/supabase'
// supabase is null when env vars are missing — always guard:
if (!supabase) { /* fallback to local state */ return }
```

### Image uploads

```typescript
import { uploadImage } from '../../lib/boardService'
const url = await uploadImage(file, userId)  // returns public URL or null
```

## 10. Common Mistakes to Avoid

| Mistake | Correct approach |
|---------|-----------------|
| Using `useState` for widget data | Use `updateWidgetData(widget.id, {...})` |
| Adding `position: absolute` to widget root | WidgetWrapper handles positioning |
| Using `react-draggable` | It's removed. Use pointer events or WidgetWrapper |
| Committing directly to `main` | Always use feat/* → develop → main |
| English UI text | All visible text must be Norwegian |
| Missing DEFAULT_SIZES entry | Every WidgetType needs a DEFAULT_SIZES entry |
| Calling `updateWidget` for data changes | Use `updateWidgetData` for `widget.data` fields |
| Not guarding `supabase` for null | Always: `if (!supabase) return` |
